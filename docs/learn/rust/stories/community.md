# The Rust Community

Rust is known not just for its technical merits but for its uniquely welcoming and well-organised community.

## Prerequisites

- [History](history.md) — Rust's origins and evolution

## Governance

Rust is governed by a set of **teams** with specific responsibilities:

- **Core Team** — Overall project direction, leadership, and cross-cutting decisions
- **Language Team** — Language design, syntax, and semantics
- **Compiler Team** — The compiler implementation, LLVM integration, codegen
- **Library Team** — The standard library and its APIs
- **Cargo Team** — The build system, package manager, crates.io
- **Moderation Team** — Code of Conduct enforcement and community safety
- **Foundation Teams** — Infrastructure, community, documentation, and more

Decisions are made through **RFCs** (Request For Comments) — a transparent process where anyone can propose changes, and the community discusses them before the relevant team decides.

## Code of Conduct

Rust adopted a **Code of Conduct** early in its history, setting a standard for inclusive and respectful behaviour. The CoC applies to:

- GitHub repositories (rust-lang organisation)
- Official chat channels (Discord, Zulip)
- Conferences and meetups
- Community spaces (users.rust-lang.org, internals.rust-lang.org)

The Moderation Team handles reports and enforces the CoC — a practice that was pioneering in open source at the time.

## Community Milestones

- **2014 — RustBridge** — Workshops to teach Rust to underrepresented groups
- **2016 — first RustConf** — The community had grown enough for a dedicated conference
- **2017 — This Week in Rust** — Weekly newsletter documenting ecosystem developments
- **2018 — Rust Survey** — First annual community survey, now a key planning tool
- **2020 — Rust Foundation** — Independent non-profit foundation to steward the project
- **2024 — Rust enters TIOBE index top 20** — Reflecting growing adoption
