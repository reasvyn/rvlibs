# Code Coverage

Measuring how much of your code is exercised by tests — line, function, and region coverage.

## Prerequisites

- [Why Test](../basics/why-test.md) — basic understanding of testing
- [CI Integration](ci-integration.md) (recommended) — CI pipelines


## Running Coverage

rvtest has a self-contained coverage system — no external tools needed:

```bash
cargo rvtest --coverage
```

Output:
```
Coverage: 48.7% lines, 56.1% functions, 48.7% regions
```

## Coverage Formats

```bash
# Summary (default)
cargo rvtest --coverage

# HTML report (interactive, open in browser)
cargo rvtest --coverage --coverage-format html --coverage-open

# CI-friendly formats
cargo rvtest --coverage --coverage-format json
cargo rvtest --coverage --coverage-format lcov
cargo rvtest --coverage --coverage-format cobertura

# Set minimum threshold (fails if below)
cargo rvtest --coverage --coverage-min 80
```

## How It Works

rvtest tries multiple strategies for coverage collection, falling back as needed:

1. **`cargo-llvm-cov`** — best output quality if installed
2. **LLVM tools** — `llvm-profdata` + `llvm-cov` if available
3. **Self-contained parser** — pure Rust `.profraw` parser, no external deps
4. **Sampler** — `ptrace` + `addr2line`, Linux only

## Interpreting Coverage

| Coverage % | Meaning |
|-----------|---------|
| < 50% | Significant untested code. Prioritise critical paths. |
| 50–75% | Moderate coverage. Core functionality is tested. |
| 75–90% | Good coverage. Most code paths are exercised. |
| > 90% | Excellent coverage. Edge cases may still lurk. |

Coverage is a metric, not a goal. 100% coverage doesn't mean bug-free code, but low coverage is a strong signal that testing is insufficient.

## Glossarium

| Term | Definition |
|------|------------|
| Line Coverage | Percentage of source lines executed during tests. |
| Function Coverage | Percentage of functions that were called during tests. |
| Region Coverage | Percentage of code regions (branches, basic blocks) executed. |
| `.profraw` | Raw profile data format generated by LLVM's instrumentation. |
| Profdata | Merged, indexed profile data (intermediate format). |
