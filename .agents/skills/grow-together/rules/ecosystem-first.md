# Ecosystem-First

Teach Rust, not rvlibs. rvlibs is a vehicle, not the destination.

## Why

Learners need to navigate the real Rust ecosystem — crates.io, docs.rs, the Rust
Book, rustc, clippy, rust-analyzer. rvlibs crates are tools, not the curriculum.
The curriculum is Rust itself.

## Guidelines

- Every document must link to at least one external Rust resource (Rust Book,
  docs.rs, crates.io, Rust by Example, rustc book, etc.).
- Present rvlibs APIs in the context of broader Rust patterns. Start with how
  something works in std or the ecosystem, then show rvlibs.
- Acknowledge alternatives. Do not present rvlibs as the only or best way.
- Use `https://doc.rust-lang.org/stable/...` not `nightly` unless the feature is
  nightly-only.
- Link to the canonical community resource (Rust Book for concepts, docs.rs for
  APIs, crates.io for packages).
